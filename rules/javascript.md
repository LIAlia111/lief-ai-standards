# JavaScript 编码规范

> 来源：[Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript)（148k★）

## 变量声明

- 默认用 `const`；需要重赋值时用 `let`；永远不用 `var`
- `const` 声明统一放一块，`let` 声明统一放一块
- 每个声明一行，不用逗号分隔的多变量声明

## 对象

- 用字面量语法创建对象：`{}` 而非 `new Object()`
- 用计算属性名创建含动态键的对象
- 对象方法用简写语法；属性值与变量名同名时用属性值简写
- 简写属性集中放在对象声明开头
- 只对不合法标识符的属性名加引号
- 用 `Object.hasOwn()` 替代直接 `.hasOwnProperty()`
- 浅拷贝对象用展开语法 `{...obj}`，不用 `Object.assign()`

## 数组

- 用字面量创建数组：`[]` 而非 `new Array()`
- 复制数组用展开语法 `[...arr]`，不用循环
- 将类数组对象转为数组用 `Array.from()`

## 解构

- 访问对象多个属性时用对象解构
- 数组取值用数组解构
- 函数返回多值时用对象解构，不用数组解构

## 字符串

- 字符串用单引号 `''`；模板字面量仅用于插值或换行
- 不要跨行截断长字符串；用模板字面量动态拼接字符串
- 永远不用 `eval()` 处理字符串

## 函数

- 用命名函数表达式替代函数声明，避免 hoisting 问题
- 立即调用函数表达式两侧都用括号包裹
- 不要在非函数块（if/while）内声明函数；改用变量赋值
- 不要将参数命名为 `arguments`；用 rest 语法 `...args` 替代
- 用默认参数语法，不要在函数体内 mutate 或判断参数
- 默认参数放在参数列表末尾
- 永远不用 `Function` 构造函数（等价于 `eval()`）

## 箭头函数

- 行内回调用箭头函数，提供正确的 `this` 绑定
- 函数体为单表达式时省略花括号并隐式 `return`
- 多行箭头函数表达式用括号包裹
- 箭头函数参数始终加括号（即使只有一个参数）

## 类

- 用 `class` 语法；不直接操作 `prototype`
- 用 `extends` 实现继承，保证正确的 `instanceof` 行为
- 方法可 `return this` 实现链式调用
- 不写空构造函数；类中不需要 `constructor` 时可省略
- 不要重复声明类成员
- 类方法要用到 `this` 或声明为 `static`，否则该方法应提为模块函数

## 模块

- 始终用 `import/export`，不用非标准模块系统
- 不用通配符导入；明确指定命名导出或默认导出
- 不在 `import` 行直接 `export`；先 import 再单独 export
- 每个路径只 `import` 一次
- 不导出可变绑定；只导出常量或默认函数/类
- 单一导出的模块优先用默认导出
- 所有 `import` 语句放在文件最顶部

## 比较与控制流

- 所有比较用 `===` 和 `!==`，不用 `==` 和 `!=`
- 布尔值用快捷判断；字符串和数值显式比较
- 用高阶函数（`map`、`filter`、`reduce`）替代 `for` 循环
- 属性访问用点号；变量键访问用方括号

## 格式

- 缩进 2 空格；多行结构加尾逗号；始终写分号
